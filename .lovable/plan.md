## Goal

Transform the AgriLink dashboard into an interactive, hackathon-worthy experience with role-specific data visualizations, geospatial intelligence (weather + soil), market analytics, and an order/bidding insights layer.

## Exploration summary

- Stack: TanStack Start + React + Tailwind + shadcn. `recharts` chart wrappers already exist (`src/components/ui/chart.tsx`).
- Roles: `farmer`, `buyer`, `transporter` (from `useAuth`). Existing dashboard at `src/routes/_authenticated.dashboard.tsx` already branches by role — perfect place to inject widgets.
- Tables available: `products`, `bids`, `orders`, `logistics`, `profiles`, `user_roles`. Enough to derive market trends, win-rates, demand, order pipelines.
- AI gateway already wired (`LOVABLE_API_KEY`) — can power natural-language insights.

## What we'll build (role-aware interactive dashboard)

### 1. Farmer dashboard — "Grow smarter"

- **Weather + Field Map widget** (Leaflet + OpenWeather free API via edge function)
  - Interactive map centered on farmer's `profiles.location` (geocoded)
  - Weather overlay: 7-day forecast cards (temp, rain %, humidity, wind)
  - "Best harvest window" badge generated by AI from forecast
- **Soil Health card**
  - Inputs: pH, moisture, N-P-K (manual entry, stored in new `soil_reports` table)
  - AI-generated recommendations (which crops thrive, fertilizer tips) via new `soil-advisor` edge function
  - Radar chart of soil nutrients
- **My Crop Performance**
  - Bar chart: price/unit per listing vs district average
  - Demand-score sparkline over time

### 2. Buyer dashboard — "Bid with confidence"

- **Market Price Trends**
  - Line chart of average `price_per_unit` per category over last 30/90 days (from `products` + `bids`)
  - Filter by crop / region
- **My Bidding Stats**
  - KPIs: win rate, avg bid vs winning bid, total spend
  - Stacked bar: pending / accepted / rejected bids per month
- **Top Suppliers near me** (mini map with farmer pins)
- **AI bid coach** — uses existing `suggest-bid` function, surfaced as a "Smart suggestion" panel

### 3. Transporter dashboard — "Optimize routes"

- **Live Jobs Map** — pickup pins for unassigned orders, color-coded by value
- **Earnings chart** — completed orders bar chart (week/month)
- **Route distance vs payout scatter** (uses existing `optimize-route` function)

### 4. Shared "Market Pulse" tab (visible to all roles)

- **Bargraph: product category vs avg price** (national snapshot)
- **Companies / buyers leaderboard** — top buyers by order volume
- **Bidding heatmap** — crop × region intensity grid
- **Live ticker** of recent accepted bids (Supabase Realtime on `bids` table)

## Technical plan

### New dependencies

- `react-leaflet` + `leaflet` for maps (lightweight, Worker-compatible)
- Reuse existing `recharts` via `chart.tsx`

### New edge functions (server functions via `createServerFn`)

- `get-weather` — proxies OpenWeather (needs `OPENWEATHER_API_KEY` secret)
- `geocode-location` — converts profile location text → lat/lng (OpenWeather geocoding, same key)
- `soil-advisor` — Lovable AI (`google/gemini-2.5-flash`) → recommendations from soil inputs
- `market-trends` — aggregates `products`/`bids` into time-series JSON

### Database changes (migration)

- New `soil_reports` table: `id, farmer_id, ph, moisture, nitrogen, phosphorus, potassium, notes, created_at` + RLS (farmer manages own, public read for AI insights opt-in)
- Add `latitude`, `longitude` columns to `profiles` (nullable) for map pinning
- Enable Realtime on `bids` for the live ticker

### UI structure

- Refactor `_authenticated.dashboard.tsx` into tabbed layout using existing `tabs.tsx`:
  - `Overview` (current cards) | `Map & Weather` | `Analytics` | `Market Pulse`
- New components in `src/components/dashboard/`:
  - `WeatherMap.tsx`, `SoilHealthCard.tsx`, `MarketTrendsChart.tsx`, `BidStatsChart.tsx`, `JobsMap.tsx`, `LiveBidTicker.tsx`, `TopBuyersLeaderboard.tsx`

### Secret needed

- `OPENWEATHER_API_KEY` (free tier covers weather + geocoding) — will request via `add_secret` after approval.

## Build order

1. Migration: `soil_reports` table + `profiles.lat/lng` + realtime on `bids`
2. Install `react-leaflet` + `leaflet`
3. Request `OPENWEATHER_API_KEY`
4. Edge functions: `get-weather`, `geocode-location`, `soil-advisor`, `market-trends`
5. Tabbed dashboard refactor + 4 farmer widgets
6. Buyer widgets (charts + bid stats + supplier map)
7. Transporter widgets (jobs map + earnings)
8. Shared Market Pulse tab + realtime ticker
9. End-to-end QA per role
